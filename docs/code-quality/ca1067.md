# <a name="ca1067-override-equals-when-implementing-iequatable"></a>CA1067: invalidar Equals al implementar IEquatable

|||
|-|-|
|TypeName|EquatableAnalyzer|
|Identificador de comprobación|CA1067|
|Category|Microsoft. Design|
|Cambio importante|Poco problemático|

## <a name="cause"></a>Causa

Un tipo implementa <xref:System.IEquatable%601>, pero no invalida <xref:System.Object.Equals%2A> el método.

## <a name="rule-description"></a>Descripción de la regla

Un tipo que <xref:System.IEquatable%601> implementa una interfaz indica que admite la comparación de dos instancias del tipo para determinar si son iguales. También debe invalidar las implementaciones de la clase <xref:System.Object.Equals%2A> base <xref:System.Object.GetHashCode> de los métodos y para que su comportamiento sea coherente con <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> el de la implementación. Consulte [aquí](/dotnet/api/system.iequatable-1#notes-to-implementers) para más información.

La <xref:System.Object.Equals%2A> implementación de debe devolver resultados coherentes con <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> la implementación de.

## <a name="how-to-fix-violations"></a>Cómo corregir infracciones

Para corregir una infracción, <xref:System.Object.Equals%2A> invalide e implemente la <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> invocación de la implementación. Por ejemplo, los dos fragmentos de código siguientes muestran una infracción de la regla y cómo corregirla:

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```

## <a name="when-to-suppress-warnings"></a>Cuándo suprimir advertencias

No se suprimen las infracciones de esta regla.
