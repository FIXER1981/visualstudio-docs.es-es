# <a name="receive-activity-designer"></a>Diseñador de actividades Receive
El diseñador de actividad **De** recepción <xref:System.ServiceModel.Activities.Receive> se usa para crear y configurar una actividad. Una actividad <xref:System.ServiceModel.Activities.Receive> es una actividad que recibe un mensaje que puede ser de tipo integrado como <xref:System.ServiceModel.Channels.Message>, <xref:System.IO.Stream> o <xref:System.Xml.Linq.XElement> o bien, un contrato de datos definido por la aplicación, contrato de mensaje o clase XML que se pueden serializar.

## <a name="the-receive-activity"></a>Actividad Receive
 La actividad <xref:System.ServiceModel.Activities.Receive> puede recibir un elemento único o varios elementos, en función del tipo de contenido de Receive que se utilice. Una actividad <xref:System.ServiceModel.Activities.SendReply> se puede enlazar a una actividad <xref:System.ServiceModel.Activities.Receive> que reciba un mensaje como parte de un patrón de intercambio de mensajes solicitud/respuesta en el servicio.

### <a name="using-the-receive-activity-designer"></a>Utilizar el diseñador de actividades Receive
 El **Receive** Diseñador de actividades de recepción se puede encontrar en la categoría **Mensajería** [!INCLUDE[wfd2](../includes/wfd2-md.md)] del Cuadro de **herramientas**, a la que se tiene acceso haciendo clic en la pestaña **Cuadro** de herramientas de la (Alternativamente, seleccione Barra de **herramientas** en el menú **Ver** o CTRL+ALT+X.)

 El Diseñador de actividades de **recepción** se puede [!INCLUDE[wfd2](../includes/wfd2-md.md)] arrastrar desde el cuadro de **herramientas** y colocar en la superficie donde se suelen colocar las actividades. Esto crea una actividad <xref:System.ServiceModel.Activities.Receive> con una propiedad <xref:System.Activities.Activity.DisplayName%2A> predeterminada de Receive. El <xref:System.Activities.Activity.DisplayName%2A> se puede editar en el encabezado de la **recepción** diseñador de actividad o en el **DisplayName** cuadro de la cuadrícula de propiedades.

 Para crear <xref:System.ServiceModel.Activities.SendReply> una actividad y <xref:System.ServiceModel.Activities.Receive> enlazarla a la actividad seleccionada, haga clic con el botón secundario en el Diseñador de actividades **de recepción,** haga clic en el elemento **Crear SendReply** en el menú contextual y el diseñador **SendReplyToReceive** aparece debajo del diseñador **De recepción.** La actividad <xref:System.ServiceModel.Activities.SendReply> es una actividad que envía el mensaje de respuesta como parte de un patrón de intercambio de mensajes solicitud/respuesta en el servicio. Se puede configurar con el diseñador **SendReplyToReceive.**

 Como alternativa, el **ReceiveAndSendReply** diseñador de plantillas en el **mensajería** categoría del <xref:System.ServiceModel.Activities.Receive> cuadro <xref:System.ServiceModel.Activities.SendReply> de **herramientas** se puede usar para crear un par de preconfigurado y actividad. [!INCLUDE[crabout](../includes/crabout-md.md)]el uso de la **ReceiveAndSendReply** y **SendReplyToReceive** plantilla, vea el [ReceiveAndSendReply](../workflow-designer/receiveandsendreply-template-designer.md) tema.
