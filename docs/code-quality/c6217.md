# <a name="c6217"></a>C6217

C6217 de ADVERTENCIA: Conversión implícita entre tipos enteros semánticamente diferentes: probando HRESULT con ' not '. Considere la posibilidad de usar en su lugar la macro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) o [failed](/windows/desktop/api/winerror/nf-winerror-failed) .

Esta advertencia indica que se está probando un `HRESULT` con el operador not (`!`). Un valor de 0 indica un éxito (`S_OK`) en `HRESULT`. Sin embargo, 0 indica error en un tipo booleano. La prueba de `HRESULT` con el operador not (`!`) para determinar qué bloque de código ejecutar puede provocar que se siga la ruta de acceso de código incorrecta. Esto dará lugar a resultados no deseados.

## <a name="example"></a>Ejemplo

El código siguiente genera esta advertencia porque el operador not se usa para determinar si un valor `HRESULT` se ha realizado correctamente o no. En este caso, se ejecuta una ruta de acceso de código incorrecta porque `( !hr )` ejecuta el código de error:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Para corregir esta advertencia, el código siguiente usa la macro `FAILED` para buscar el error:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Para esta advertencia, el tipo `SCODE` es equivalente a `HRESULT`.

El valor de éxito típico de HRESULT (`S_OK`) es `false` cuando se prueba como booleano.

Para comprobar si `HRESULT` es un éxito, utilice en su lugar la macro `SUCCEEDED`.
