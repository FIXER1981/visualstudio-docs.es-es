# <a name="c6284"></a>C6284
ADVERTENCIA C6284: objeto pasado como parámetro '% d ' cuando se requiere una cadena en la llamada a \<function >.

 Esta advertencia indica que la cadena de formato especifica una cadena, por ejemplo, una especificación `%s` para `printf` o `scanf`, pero C++ se ha pasado un objeto en su lugar.

 Este defecto podría producir resultados incorrectos o bloqueos.

 Este mensaje suele aparecer debido a que se pasa C++ un objeto que implementa algún tipo de cadena, por ejemplo `std::string`, `CComBSTR` o `bstr_t`, a una llamada de estilo de C @no__t -4. En función de la implementación de la C++ clase, es decir, si se definen los operadores de conversión adecuados C++ , a menudo se pueden usar de forma transparente los objetos de cadena siempre que se requieran cadenas de C. sin embargo, dado que los parámetros para @no__t funciones de estilo -2 son esencialmente sin tipo, no se produce ninguna conversión a una cadena.

 Dependiendo del objeto, podría ser adecuado insertar un operador `static_cast` en el tipo de cadena adecuado, por ejemplo, `char *` o `TCHAR``*`, o bien llamar a una función miembro que devuelva una cadena, por ejemplo, `c_str()`, en instancias de `std::string`.

## <a name="example"></a>Ejemplo
 El código siguiente genera esta advertencia porque se pasa un `CComBSTR` a la función `sprintf`:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 En el código siguiente se usa la conversión estática para corregir esta ADVERTENCIA:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
