# <a name="c6276"></a>C6276
C6276 de ADVERTENCIA: Conversión entre tipos de cadena semánticamente diferentes: char * en wchar_t @ no__t-0. El uso de una cadena no válida puede dar lugar a un comportamiento indefinido

 Esta advertencia indica una conversión potencialmente incorrecta de una cadena ANSI (`char_t*`) a una cadena Unicode (`wchar_t *`). Dado que las cadenas Unicode tienen un tamaño de carácter de 2 bytes, esta conversión podría producir cadenas que no terminan correctamente. El uso de estas cadenas con la biblioteca WCS * de funciones podría producir saturaciones de búfer e infracciones de acceso.

## <a name="example"></a>Ejemplo
 El código siguiente genera esta advertencia:

```
#include <windows.h>
VOID f()
{
   WCHAR szBuffer[8];
   LPWSTR pSrc;

   pSrc = (LPWSTR)"a";
   wcscpy(szBuffer, pSrc);
}
```

 El código siguiente corrige esta advertencia anexando la letra L para representar el carácter ASCII como un carácter ancho:

```
#include <windows.h>

VOID f()
{
   WCHAR szBuffer[8];
   LPWSTR pSrc;

   pSrc = L"a";
   wcscpy(szBuffer, pSrc);
}
```

 En el código siguiente se usa la función de manipulación de cadenas seguras, `wcscpy_s`, para corregir esta ADVERTENCIA:

```
#include <windows.h>

VOID f()
{
   WCHAR szBuffer[8];
   LPWSTR pSrc;
   pSrc = L"a";
   wcscpy_s(szBuffer,8,pSrc);
}
```
