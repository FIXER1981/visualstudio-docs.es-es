# <a name="c6273"></a>C6273
ADVERTENCIA 6273: valor no entero pasado como parámetro \<number > cuando se requiere un entero en la llamada a \<Function >: si se pasa un valor de puntero, se debe usar% p

 Esta advertencia indica que la cadena de formato especifica un entero, por ejemplo, una especificación de `%d`, longitud o precedencia para `printf`, pero se pasa un valor no entero como un `float`, una cadena o `struct` como parámetro. Es probable que este defecto genere resultados incorrectos.

## <a name="example"></a>Ejemplo
 El código siguiente genera esta advertencia porque se requiere un entero en lugar de un `float` a `sprintf`Function:

```
#include <stdio.h>
#include <string.h>

void f_defective()
{
  char buff[50];
  float f=1.5;

  sprintf(buff, "%d",f);
}
```

 En el código siguiente se usa una conversión de enteros para corregir esta ADVERTENCIA:

```
#include <stdio.h>
#include <string.h>

void f_corrected()
{
  char buff[50];
  float f=1.5;

  sprintf(buff,"%d",(int)f);
}
```

 En el código siguiente se usa la función de manipulación de cadenas seguras, `sprintf_s`, para corregir esta ADVERTENCIA:

```
#include <stdio.h>
#include <string.h>

void f_safe()
{
  char buff[50];
  float f=1.5;

  sprintf_s(buff,50,"%d",(int)f);
}
```

 Esta advertencia no es aplicable en Windows 9x y Windows NT versión 4 porque% p no es compatible con estas plataformas.
