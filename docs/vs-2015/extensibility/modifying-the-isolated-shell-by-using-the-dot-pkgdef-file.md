# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Modificación del Shell aislado mediante el archivo .Pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

El archivo. pkgdef admite valores que puede usar para personalizar una aplicación de Shell aislado. Especifica los valores que se crean cuando se instala una aplicación en un equipo y al que hace referencia el shell de Visual Studio cuando se inicia la aplicación. La configuración se organiza en el archivo en función de las claves del registro correspondientes.  

> [!WARNING]
> Tenga en cuenta que los archivos. pkgdef que no se declaran en el archivo. vsixmanifest del VSPackage no se examinan cuando se inicia Visual Studio.  

 El archivo. pkgdef contiene secciones que se identifican mediante una clave, ya sea `[$RootKey$]` o `[$RootKey$\` *subclave* `]` , donde $RootKey $ es la clave raíz de la aplicación.  

 Cada sección contiene pares de nombre/valor que tienen el siguiente formato: valor de `"` *ValueName* `"=` *Value*.  

 Los valores son una cadena entre comillas o un entero de 32 bits que se representa como un valor DWORD. Los valores tienen las restricciones siguientes:  

- Todos los valores DWORD están en formato hexadecimal, por ejemplo `dword:00000001` .  

   Para los valores booleanos, 1 representa true y 0 representa false.  

- Todas las cadenas GUID están en formato de registro, por ejemplo, `"{00000000-0000-0000-0000-000000000000}"` .  

- Todos los identificadores de recursos localizables tienen el formato "@*resourceId*" o "#*resourceId*", donde *resourceId* es el identificador de recursos del paquete de interfaz de usuario de la aplicación, por ejemplo, `"@102"` . El paquete de interfaz de usuario de la aplicación es el paquete al que se hace referencia en la configuración AppLocalizationPackage.  

  Por ejemplo,  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Puede agregar comentarios al archivo. pkgdef. Un Comentario de una sola línea tiene dos barras diagonales como los dos primeros caracteres.  

 Para obtener una lista de las cadenas de sustitución, vea [cadenas de sustitución usadas en. Pkgdef y. Archivos Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 En las secciones siguientes se describen los valores específicos del registro que afectan al comportamiento del shell de Visual Studio en modo aislado. También puede definir valores de registro adicionales para la aplicación en este archivo.  

> [!NOTE]
> Si no se proporciona un valor en el archivo. pkgdef, no se realiza ninguna entrada correspondiente en el registro.

## <a name="binding-path-settings"></a>Configuración de la ruta de acceso de enlace  
 La clave [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] contiene la lista de directorios que el shell comprueba para los ensamblados. Estos directorios se agregan a la lista de directorios en los que el shell sondea los ensamblados privados para la aplicación.  

 De forma predeterminada, no se agregan entradas de ruta de acceso de enlace al archivo. pkgdef. Sin embargo, los siguientes subdirectorios del directorio de instalación de Visual Studio se agregan automáticamente a la lista de rutas de acceso de enlace de la aplicación en el registro.  

- Common7\IDE\  

- Common7\IDE \\ \PrivateAssemblies  

- Common7\IDE \\ \PublicAssemblies  

  Para agregar un directorio a la ruta de acceso de enlace, agregue una entrada con el formato "*directoryName*" = "", donde *directoryName* es una ruta de acceso absoluta. Por ejemplo,  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Configuración del elemento de menú paquete  
 La clave del registro [$RootKey $ \Menus] define los archivos de recursos de la interfaz de usuario para la aplicación.  

 Los valores de los elementos de menú tienen el formato "{*vsUiPackageGuid*}" = ", *resourceId*, *versionNumber*", donde *vsUiPackageGuid* es el GUID del paquete de interfaz de usuario de la aplicación, *resourceId* es el identificador de recursos del recurso CTMENU que contiene los elementos de la interfaz de usuario y *versionNumber* es un número de versión virtual para el recurso CTMENU. Para obtener más información, vea [registrar controladores de comandos de ensamblado de interoperabilidad](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 De forma predeterminada, se crea una entrada de elemento de menú en el archivo. pkgdef para el paquete de interfaz de usuario de la aplicación.  

 Para cada paquete que proporciona elementos de menú y que se distribuye como parte de la aplicación, agregue una entrada de elemento de menú para el paquete.
