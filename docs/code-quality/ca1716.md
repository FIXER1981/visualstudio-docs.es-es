# <a name="ca1716-identifiers-should-not-match-keywords"></a>CA1716: Los identificadores no deberían coincidir con palabras clave

|||
|-|-|
|TypeName|IdentifiersShouldNotMatchKeywords|
|Identificador de comprobación|CA1716|
|Categoría|Microsoft.Naming|
|Cambio importante|Problemático|

## <a name="cause"></a>Causa

El nombre de un espacio de nombres, tipo o miembro virtual o de interfaz coincide con una palabra clave reservada en un lenguaje de programación.

De forma predeterminada, esta regla solo examina los espacios de nombres, tipos y miembros visibles externamente, pero puede configurar los tipos de [visibilidad](#api-surface) y [símbolos.](#analyzed-symbol-kinds)

## <a name="rule-description"></a>Descripción de la regla

Los identificadores de espacios de nombres, tipos y miembros virtuales y de interfaz no deben coincidir con las palabras clave definidas por lenguajes que tienen como destino Common Language Runtime. Según el lenguaje que se use y la palabra clave, los errores y ambiguedades del compilador pueden dificultar el uso de la biblioteca.

Esta regla comprueba las palabras clave en los siguientes idiomas:

- Visual Basic
- C#
- C++/CLI

La comparación sin distinción entre mayúsculas y minúsculas se usa para las palabras clave de Visual Basic y la comparación entre mayúsculas y minúsculas se usa para los demás lenguajes.

## <a name="how-to-fix-violations"></a>Cómo corregir infracciones

Seleccione un nombre que no aparezca en la lista de palabras clave.

## <a name="when-to-suppress-warnings"></a>Cuándo suprimir las advertencias

Puede suprimir una advertencia de esta regla si está convencido de que el identificador no confundirá a los usuarios de la API y de que la biblioteca se puede usar en todos los idiomas disponibles en .NET.

## <a name="configurability"></a>Configuración

### <a name="api-surface"></a>Superficie Api

Si está ejecutando esta regla desde [analizadores FxCop](install-fxcop-analyzers.md) (y no con análisis heredados), puede configurar en qué partes del código base ejecutar esta regla, en función de su accesibilidad. Por ejemplo, para especificar que la regla se ejecute solo en la superficie de API no pública, agregue el siguiente par clave-valor a un archivo .editorconfig del proyecto:

```ini
dotnet_code_quality.ca1716.api_surface = private, internal
```

Puede configurar esta opción solo para esta regla, para todas las reglas o para todas las reglas de esta categoría (Naming). Para obtener más información, consulte [Configurar analizadores FxCop](configure-fxcop-analyzers.md).

### <a name="analyzed-symbol-kinds"></a>Tipos de símbolos analizados

Puede configurar los tipos de miembros que se analizarán mediante esta regla. Utilice uno o varios `Namespace` `NamedType`de `Method` `Property`estos `Event` `Parameter` valores, , , , , , como una lista separada por comas. Tenga en cuenta que el valor predeterminado incluye todos los miembros anteriores.

```ini
dotnet_code_quality.CA1716.analyzed_symbol_kinds = Namespace, NamedType, Method, Property, Event
```
