# <a name="c26112"></a>C26112
C26112 de ADVERTENCIA: El autor de la llamada no puede contener ningún bloqueo antes de llamar a \<func >.

 La anotación `_Requires_no_locks_held_` impone una condición previa de que el llamador no debe mantener ningún bloqueo mientras llama a la función. Warning C26112 se emite cuando una función no libera todos los bloqueos antes de llamar a otra función.

## <a name="example"></a>Ejemplo
 En el ejemplo siguiente se genera la advertencia C26112 porque la llamada a `NoLocksAllowed` en la sección bloqueada infringe la condición previa `_Requires_no_locks_held_`.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

_Requires_no_locks_held_

void NoLocksAllowed(DATA* p)
{
     // Lock sensitive operations here
}

void LocksHeldFunction(DATA* p)
{
    EnterCriticalSection(&p->cs);
    NoLocksAllowed(p); // Warning C26112
    LeaveCriticalSection(&p->cs);
}
```
