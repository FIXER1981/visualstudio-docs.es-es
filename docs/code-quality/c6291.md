# <a name="c6291"></a>C6291
C6291 de ADVERTENCIA: Operación bit a bit en el resultado lógico:! tiene mayor precedencia que &#124;. Use &#124; &#124; o (! ( x &#124; y)) en su lugar

El operador `!` produce un resultado booleano y el operador `|` (OR bit a bit) toma dos argumentos aritméticos. El operador `!` también tiene mayor prioridad que `|`.

Por lo tanto, se ha detectado uno de los siguientes errores:

- La expresión se escribe entre paréntesis:

   Dado que el resultado de `!` es un valor booleano (cero o uno), un intento de probar que dos variables tienen bits establecido solo finalizará la prueba de que el bit más bajo está presente en el lado derecho: `((!x) | y) != (!(x | y))` cuando `x == 0` y `y == 1`.

- El operador `!` es incorrecto y debe ser `~` en su lugar:

   El operador `!` tiene un resultado booleano, pero el operador `~` tiene un resultado aritmético. Estos operadores nunca son intercambiables, aunque operen en un valor booleano (cero o uno): `((!x) | y) != ((~x) | y)` cuando `x == 1` y `y == 0`.

- El operador binario `|` es incorrecto y, en su lugar, debe ser `||`:

   Aunque `|` a veces se puede intercambiar con `||`, no es equivalente porque fuerza la evaluación del lado derecho de la expresión. Algunos efectos secundarios en este tipo de expresión pueden ser terminal: `(!p | (*p == '\0'))`, cuando `p == NULL`, se debe desreferenciar para evaluar la otra mitad de la expresión.

Esta advertencia no se indica si el operador `!` está en el lado derecho del operador `|` porque este caso suele ser solo el caso relativamente inofensivo de un operador incorrecto.

Es difícil juzgar la gravedad de este problema sin examinar el código. Se debe inspeccionar el código para asegurarse de que se está produciendo la prueba prevista.

Esta advertencia siempre indica una posible confusión en el uso de una precedencia de operador o operador.

## <a name="example"></a>Ejemplo

El código siguiente genera esta advertencia:

```cpp
void f(int x, int y )
{
  if (!x | y)
  {
    //code
  }
}
```

Para corregir esta advertencia, use uno de los ejemplos que se muestran en el código siguiente:

```cpp
void fC(int x, int y )
{
  /* When checking whether any bits are set in either x or y. */
  if (!(x | y))
  {
    // code
  }
  /* When checking whether bits are set in either */
  /* the complement of x or in y. */
  if ((~x) | y)
  {
    // code
  }
}

#include <windows.h>
void f(int x, BOOL y )
{
  /* When y is a Boolean or Boolean result. */
  if ((!x) || y)
  {
    // code
  }
}
```
